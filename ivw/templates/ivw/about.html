About This Project

This web application is developed as part of a Computer Science Internal Assessment (IA). It is designed to assist Instituto Voar, from Prima Inova, in organizing and categorizing stakeholder demands effectively.

The system allows Francisleide Pustilnik and her team to manage demands following the AA1000 Accountability Standards, ensuring proper classification, prioritization, and tracking of programs aimed at addressing these demands.

Below is the success criteria that the system must fulfill:

Category Success Criteria
Minimum
  • Access to the system is limited to my client, her team, and any other user specified by her, using a login system with hashed password storage for security.
  • Users can register and edit stakeholders, materiality principles outlined by the AA1000 standards, the United Nation’s SDGs, and programs executed by “Instituto Voar” in response to demands.
  • Users can register and edit new demands, specifying its title and description, while the system automatically stores its date of submission.
  • Users can categorize demands by associating them with one or more stakeholders, materiality principles, and SDGs.
  • Users can rate demands on a 1-5 scale for potential impact, implementation efforts, and beneficiaries. The average of these ratings should be automatically stored by the system under the “significance” field.
  • After creation, demands are automatically classified as “awaiting prioritization,” but users can change the status of a demand as “approved” or “canceled.”
Desirable
  • Users can create action plans for implementing each demand listed as “approved,” specifying the action plans’ titles, start and end dates, responsible persons, and estimated costs.
  • The system allows associating approved demands with response programs executed by “Instituto Voar.”
  • The system includes a dashboard with the number of demands associated with each materiality issue, SDG, and stakeholder.
  • The system includes a dashboard illustrating the number of demands under each of the three statuses of “approved,” “canceled,” or “awaiting prioritization.”
If Possible
  • The system contains a table that automatically calculates the total estimated cost per program, based on the demands and their action plans it responds to. The table shows both ongoing and planned expenses for each program.
  • The system enables stakeholders to insert their demands directly to its database, thus avoiding my client and her team from registering demands manually.
